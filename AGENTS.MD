# Project Agent Instructions

- Use English for all code comments.
- Prefer existing functions and normalization utilities from the project's current libraries instead of introducing custom equivalents.
- Only introduce helper functions when the functionality is reused or when the helper encapsulates a meaningful special-purpose behavior.
- Avoid fallbacks. If a fallback is unavoidable, clearly mark when it is triggered with a UI warning so development errors are immediately visible.
- Keep code lean and focused.
- Treat the Codex as the central reference for all rules information.
- Do not add migrations. The system is still in development, so existing data does not need to be migrated.
